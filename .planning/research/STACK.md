# Technology Stack

**Project:** Sedgwick County Tax Scraper
**Researched:** 2024
**Confidence:** HIGH

## Recommended Stack

### Core Framework (Scraping)
| Technology | Version | Purpose | Why |
|------------|---------|---------|-----|
| Node.js | 20+ | Execution environment | Standard, excellent ecosystem for web scraping and async operations. |
| Playwright | 1.40+ | Browser automation | Modern standard for scraping complex, dynamic SPAs and ASP.NET applications. Superior auto-waiting and cross-browser support compared to Puppeteer. |

### Core Framework (Dashboard)
| Technology | Version | Purpose | Why |
|------------|---------|---------|-----|
| Vite | 5+ | Frontend tooling | Extremely fast, zero-configuration bundler. Keeps the frontend dependency footprint minimal. |
| Vanilla JS/HTML | ES6+ | UI Logic & Structure | Avoids the overhead of heavy frameworks (React/Vue) for a simple, single-purpose dashboard. Aligns with the "light and straightforward" constraint. |

### Database
| Technology | Version | Purpose | Why |
|------------|---------|---------|-----|
| Static JSON | N/A | Data storage and transport | Generated by the scraper and served statically. Eliminates the need for a live database or backend API, drastically simplifying hosting and maintenance. |

### Infrastructure
| Technology | Version | Purpose | Why |
|------------|---------|---------|-----|
| GitHub Actions | N/A | Scheduled scraping (Cron) | Free, integrated CI/CD to run the Playwright scraper on a schedule and commit the updated JSON data. |
| Static Hosting | N/A | Dashboard Hosting | GitHub Pages or Cloudflare Pages are perfect fits for a Vite-built static site using a JSON payload. Free and highly performant. |

### Supporting Libraries
| Library | Version | Purpose | When to Use |
|---------|---------|---------|-------------|
| Chart.js | 4+ | Data Visualization | Stable, excellent Vanilla JS support. Best for standard, clean charts without complex interactions. |
| Tailwind CSS | 3.4+ | Styling | (Optional Upgrade) Highly recommended to replace Vanilla CSS for rapid, consistent styling without leaving the HTML file, keeping the lightweight ethos intact. |

## Alternatives Considered

| Category | Recommended | Alternative | Why Not |
|----------|-------------|-------------|---------|
| Scraping Framework | Playwright | Crawlee / Apify | Overkill for a single targeted site. Crawlee is better suited for large-scale crawling (millions of pages) requiring complex proxy rotation and queuing. |
| Frontend Framework | Vanilla JS | Next.js / React | Violates the project constraints of keeping dependencies light. The dashboard is currently too simple to justify a meta-framework. |
| Visualization | Chart.js | Apache ECharts | ECharts is an enterprise powerhouse meant for massive datasets (100k+ points) and complex interactions. Chart.js is sufficient and easier for basic property value tracking. |
| Data Storage | Static JSON | SQLite / DuckDB | A database requires backend API infrastructure (or WASM complexities on the frontend). Static JSON is perfect for a limited dataset (e.g., a single neighborhood's tax history). |

## Installation

```bash
# Core Scraping Dependencies
npm install playwright

# Core Dashboard Dependencies
npm install vite chart.js

# Development Dependencies
npm install -D typescript @types/node
```
